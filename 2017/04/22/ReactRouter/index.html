2.2 路由参数

通过Route路由的组件，可以拿到一个match参数，这个参数是个对象，包含以下的数据

  • isExact: 是否做全等匹配
  • params: path中包含的一些额外的数据
  • path: Route组件path属性
  • url：实际url的hash值

2.3 Router

在引用的部分，引用的是BrowserRouter。老版本其实用的就是Router，然后配置一下history。history是用来做兼容不同浏览器或者环境下的历史记录的管理，跳转或者后退，history就会去记录这些变化。之前的版本将history分为3类

  • hashHistory 老版本的浏览器
  • browserHistory h5的history
  • memoryHostory node环境下的history

4.0之前的版本实现了createHashHistory、createBrowserHistory和create MemoryHistory三个方法来创建三种情况下的history，具体不讨论了。

而在4.0版本，react-router-dom 直接将这三种做了内置，我们可以看到有3中Router，BrowserRouter、HashRouter、MemoryRouter这三种Router来替代之前的方式即可。

这里使用的是HashRouter 也就是 老版本浏览器的history。

2.4 Route标签

在上面的例子中可以看到Route标签中有以下几个prop

  • exact: 对等匹配 propType.bool
  • path: 路径 propType.string
  • component: 路径跳转之后对应的组件 propType.func
  • render: 直接在这里渲染 propType.func

这些prop都不是必选，当然如果没有path，直接用的就是默认渲染。

Route的作用就是当url和Route中path属性的值匹配的时候，就去渲染component中的内容或者render中的内容。
