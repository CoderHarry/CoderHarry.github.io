1. 垃圾回收机制

1.1 小整数对象池

整数在实际开发中是使用最多的，所以Python为了优化速度，使用了小整数对象池，避免为整数频繁申请和销毁内存空间

[-5, 257)这些整数对象都是提前建好的，不会被垃圾回收。

同理, 单个字符也是这样的，单个字符共用对象，常驻内存.

1.2 intern机制

python会不会创建8个对象么？不会。python只会占用一个’hello world’所占用的空间，通过引用计数去维护何时释放，这个机制就是 —> intern机制

默认是开启intern机制的，共用对象，引用计数为0，则会销毁。

字符串(含有空格)，不可修改，没有开启intern机制，不公用对象。

1.3 GC (Garbage collection)

python采用了和java一样的垃圾回收机制，但是不一样的是：python采用的是引用计数为主，标记-清楚和分代收集两种机制为辅的策略。

1.4 GC模块

gc模块提供一个接口给开发者设置垃圾回收的选项。单纯的采用引用计数没有处理循环引用，所以gc模块的一个主要功能就是解决循环引用。

1.4.1 常用的函数
  1. gc.set_debug(flags) 设置gc的debug日志，一般都是设置为gc.DEBUG_LEAK
  2. gc.collect([generation]) 显式进行垃圾回收，可以输入参数，0代表只检查第一代，1代表检查一、二代，2代表检查一、二、三代对象，如果不传参数，执行full collection，也就是等于传2。返回不可达对象的数目
  3. gc.get_threshold() 获取gc模块自动执行垃圾回收机制的频率
  4. gc.set_threshold(threshold0[, threshold1[, *]]) 设置频率
  5. gc.get_count() 获取当前自动执行垃圾回收机制的计数器，返回一个长度为3的列表
1.4.2 gc模块的自动垃圾回收机制

必须要import gc模块，并且设置is_enable() = True 才会启动自动垃圾回收机制。

这个机制主要做就是发现处理不可到达的垃圾对象。
