1.2. 什么是元类

元类就是用来创建类的东西。我们平常创建类的时候就是为了创建实例对象，在python中一切皆为对象，所以元类就是用来创建这些类的对象而存在的。

元类就是类的类。

type实际就是一个元类。type就是python在背后创建类的元类。通过 class 属性可以查看当前对象所属的类。

因此，元类就是创建类这种对象的东西。type就是Python的内建元类。

  2. metaclass 属性

    你可以在定义一个类的时候为其添加 metaclass 属性

    如果这么做了，Python就会用元类来创建Foo，小心点，这里面有技巧。首先写下class(object), 但是类Foo还没有在内存中创建。Python会在类的定义中寻找 metaclass 属性，如果找到，使用它创建类Foo，否则，使用内建的type来创建这个类。

    Python做了如下的操作:

    1. Foo中有 metaclass 这个属性吗？如果有，会通过 metaclass创建一个名字为Foo的类（对象）。
    2. 如果没有找到 metaclass，它会继续在父类Bar中寻找，如果找到和之前的操作一致
    3. 如果Python在父类中都找不到 metaclass， 它就会在模块层次中去寻找, 并且尝试同样的操作
    4. 如果还是找不到 metaclass ， 则使用内置的type来创建这个类对象。

    问题来了，你可以在 metaclass 中防止些什么代码呢？ 创建类的东西的代码片段。什么可以创建类呢？ type，或者任何使用到type或者子类化type的东西都可以。

3. 生成器

3.1 什么是生成器
3.1.1 列表推导
3.2.2 生成器

通过列表生产式，很容易创建一个列表，但是由于受到内存的限制，列表的容量肯定是有限制的。而且创建一个包含100万个元素的列表，占用非常大的空间，而且可能我们使用到的仅仅是前面几个元素，后面绝大多数都浪费了。所以，如果列表元素可以按照某种算法推算出来，那我们是否可以在循环的过程中不断推算出后面的元素呢？这样可以不用创建完整的list，节约大量的内存空间。在Python中，这种一边循环一边计算的机制就称之为生成器:generator。

3.2 如何创建生成器
3.2.1 创建生成器的2种方式

  1. 创建生成器，把列表生成式的[]改为()即可

    通过next()函数获得生成器的下一个返回值

    生成器保存的是算法，每次调用next(l)，就计算出l的下一个元素的值，直到计算到最后一个元素，如果没有更多的元素的时候，会跑出StopIteration的异常。当然，这种调用next()方法很变态，所以一般使用for循环来调用，因此，生成器也是可迭代对象。

  2. 通过yield方式创建生成器

    循环过程中不断调用yield，就会不断的中断。当前需要给循环一个条件用来结束循环。每次执行到yield处就会中断。

    发现使用for循环获取不到函数最后的返回值

    如何获取？手动捕获异常

3.3.3 生成器总结

生成器是这样一个函数，它记住了上一次返回时在函数体中的位置。对于生成器函数的第二次(或第n次)调用跳转至函数的中间，而上次调用的所有的局部变量都保持不变。

生成器不仅仅记住了它的数据状态，同时还记住了它在流控制构造中的位置。

生成器的特点：

  1. 节约内存
  2. 迭代到下一次调用时，所使用的参数都是第一次保留下来的，不会新创建。

4. 迭代器

4.1 什么是迭代器

迭代是访问元素的一种方式。迭代器是一个可以记住遍历位置的对象。迭代器对象从集合的第一个元素开始访问，知道元素访问结束。只前进不后退。

4.2 可迭代对象

以直接作用for循环的数据有以下几种

  1. 集合数据类型

  2. generator， 包括生成器和带yeild的generator function

    这些可以直接作用于for循环的对象统称为可迭代对象: Iterable

4.3 判断是否可迭代
4.4 迭代器

可以被next()函数调用并不断返回下一个值的对象称为迭代器: Iterator

4.5 iter()

生成器都是Iterator对象，但list，dict、str虽然都是Iterable，却不是Iterator.

把list、dict、str等Iterable编程iterator可以使用iter()函数。

4.6 总结

1. 凡是可作用于for循环的对象都是Iterable类型

2. 凡是可作用于next()函数的对象都是Iterator类型

3. 集合数据类型是Iterable却不是Iterator，可以通过iter()函数转换变为Iterator对象

6. 私有化

  1. xx：公有变量
  2. _x： 单前置下划线，私有化的属性或者方法，from somemodule import * 禁止导入，类对象和子类是可以访问的
  3. __x: 双前置下划线，私有属性，无法在外部直接访问
  4. xx: 双前后下划线, 一般是系统自带的一些属性
  5. xx_: 单后置下划线，避免与python关键字冲突
